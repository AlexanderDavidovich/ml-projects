Влияние параметра k
k = 1: очень чувствителен к шуму, может переобучаться

k = 3, 5, 7: хороший баланс между точностью и устойчивостью

Большие k: более устойчив, но может недообучаться

Преимущества и недостатки
Преимущества:

Простота реализации и понимания

Не требует предположений о распределении данных

Хорошо работает с локальными структурами в данных

Может использоваться как для классификации, так и для регрессии

Недостатки:

Чувствителен к проклятию размерности

Медленные предсказания (нужно вычислять расстояния до всех точек)

Чувствителен к масштабу признаков

Требует много памяти для хранения данных

ROC-кривая и AUC
ROC-кривая — график зависимости TPR от FPR:

TPR = Recall = TP/(TP+FN) — чувствительность

FPR = FP/(FP+TN) — доля ложных тревог

AUC (Area Under Curve) — площадь под ROC-кривой:

AUC = 1.0: идеальный классификатор

AUC = 0.5: случайный классификатор

AUC > 0.8: хороший классификатор

Задание 4.5: Собственная реализация
python
class KNearestNeighbours:
    def __init__(self, n_neighbours=5):
        self.k = n_neighbours
    
    def fit(self, X, y):
        # Ленивое обучение — просто запоминаем данные
        self.X_train = np.array(X)
        self.y_train = np.array(y)
    
    def _predict_single(self, x):
        # Вычисляем расстояния до всех точек
        distances = np.linalg.norm(self.X_train - x, axis=1)
        # Находим k ближайших
        k_indices = np.argsort(distances)[:self.k]
        # Голосование большинства
        k_nearest_labels = self.y_train[k_indices]
        most_common = Counter(k_nearest_labels).most_common(1)
        return most_common[0][0]
Принцип: Полная реализация алгоритма k-NN с нуля

8. ПРАКТИЧЕСКИЕ РЕКОМЕНДАЦИИ
Выбор k
Начинайте с k = √n (где n — размер выборки)

Всегда выбирайте нечетное k для избежания ничьих

Используйте кросс-валидацию для подбора оптимального k

Предобработка
ВСЕГДА масштабируйте данные для k-NN

Удаляйте или обрабатывайте выбросы

Рассмотрите отбор признаков при высокой размерности

Оценка модели
Используйте несколько метрик (accuracy, precision, recall, F1)

Анализируйте матрицу ошибок

Стройте ROC-кривые для полной картины

Типичные ошибки
Забыть масштабировать данные

Применить fit_transform к тестовой выборке

Не использовать кросс-валидацию для подбора k

Игнорировать дисбаланс классов

Когда использовать k-NN
Хорошо: Небольшие датасеты, локальные паттерны

Плохо: Высокая размерность, большие датасеты, много шума

ЗАКЛЮЧЕНИЕ
k-NN — простой, но мощный алгоритм классификации. Его эффективность критически зависит от правильной предобработки данных и подбора гиперпараметров. При соблюдении основных принципов k-NN показывает отличные результаты на многих практических задачах.
